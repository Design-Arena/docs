# API Overview

## Base URL

All requests use the following base URL:

```
https://www.designarena.ai/api/v1
```

## Authentication

All API requests require authentication using a Bearer token.

<CodeGroup>
```bash curl
curl -H "Authorization: Bearer sk_live_your_api_key_here" \
  https://www.designarena.ai/api/v1/leaderboard/models/website
```

```javascript JavaScript
fetch('https://www.designarena.ai/api/v1/leaderboard/models/website', {
  headers: { 'Authorization': 'Bearer sk_live_your_api_key_here' }
})
```

```python Python
requests.get(url, headers={'Authorization': 'Bearer sk_live_your_api_key_here'})
```
</CodeGroup>

### Getting an API Key

1. Apply at [designarena.ai/developers/apply](https://designarena.ai/developers/apply)
2. Describe your use case and agree to the attribution requirements
3. We'll review and email you your API key (usually 1-2 business days)

<Warning>
Keep your API key secret. Never expose it in client-side code, public repositories, or share it publicly.
</Warning>

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | [`/arenas`](/api-reference/arenas) | List all arenas and categories |
| `GET` | [`/models`](/api-reference/models) | All models with rankings across every arena/category |
| `GET` | [`/leaderboard/{arena}/{category}`](/api-reference/leaderboard) | Get leaderboard for a specific arena and category |

## Rate Limits

We do not currently enforce strict rate limits, but please be reasonable. We reserve the right to revoke keys that abuse the service.

## Caching

To ensure performance, responses are cached:
- **Models & Leaderboard** — CDN cached for **5 minutes**, stale-while-revalidate up to 10 minutes
- **Arenas** — CDN cached for **1 hour**, stale-while-revalidate up to 2 hours

## Error Codes

| Code | Meaning |
|------|---------|
| `200` | Success |
| `400` | Invalid `arena` or `category` parameter |
| `401` | Missing API key (no `Authorization` header) |
| `403` | Invalid or revoked API key |
| `500` | Server error |

### Error Response Format

```json
{
  "success": false,
  "error": {
    "code": "INVALID_API_KEY",
    "message": "The provided API key is invalid or has been revoked"
  }
}
```

## Data Volatility

Leaderboard rankings update in real-time throughout the day as new battles are recorded. Models within the same performance band may swap positions frequently. However, models that are significantly better—or in a clearly different tier—tend to remain stable within their band.

## Supported Arenas & Categories

Use `GET /arenas` to retrieve this list programmatically.

| Arena | Valid Categories |
|-------|-----------------|
| **models** | `allcategories`, `website`, `3d`, `dataviz`, `gamedev`, `uicomponent`, `logo`, `image`, `video`, `slides`, `videotovideo`, `graphicdesign`, `imagetoimage`, `svg` |
| **builders** | `website`, `mobile` |
| **agents** | `webapps`, `agon_webapps`, `fullstack` |

<Note>
`allcategories` aggregates stats across all code generation categories: `website`, `3d`, `dataviz`, `gamedev`, and `uicomponent`.
</Note>

## OpenRouter Model Mapping

Every leaderboard entry includes an `openRouterId` field that maps to OpenRouter's model identifier. This enables you to cross-reference Design Arena rankings with OpenRouter metadata (pricing, context windows, etc.).

- **`openRouterId: "anthropic/claude-opus-4.6"`** — model is available on OpenRouter
- **`openRouterId: null`** — model is not available on OpenRouter (proprietary platforms, image/video generators, deprecated models)
